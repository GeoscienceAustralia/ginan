Technology: Concepts and Techniques

This section introduces some of the principal concepts and techniques used in Precise Point Positioning. These techniques have been developed over the years since the Global Positioning System (US GPS) went live. The technology used in positioning continues to develop and improve.

Ginan is on a development journey. That development will see Ginan be released in versions incorporating new techniques and algorithms that improve:

  1. The timeliness of the analysis products i.e. reduction in latency.
  2. The accuracy of the data within analysis products.
  3. The time it takes to resolve an accurate position.

Stages in the Ginan development journey

These terms are significant because they reveal the use of specific techniques within each release of Ginan. Following is an introduction to the meaning of significant terms.

  1. The use of the term Network mode or User mode implies that Ginan has been configured as a Network Platform or User Platform respectively.
  2. Network mode refers to several stations being used together so that the observations can be used to estimate parameters i.e. produce correction products.
  3. User mode refers to station observations being processed independently of each other to produce positions.
  4. Undifferenced - observations from each reference station are processed independently, no difference is made between station observations to remove satellite clock and orbit error terms.
  5. Ionosphere-free-combination - observations obtained at different frequencies from a station are combined to remove the signal delay due to the ionosphere.
  6. Uncombined - observations are processed independently on each frequency observed. Combining frequencies is a technique used to eliminate certain factors (such as the ionosphere delay) and thus simplify equations and calculations.
  7. Ambiguity resolution (AR) - ambiguities are resolved, the number of integer phase cycles between the satellite and the receiver at first observation has been determined, thus aiding position accuracy.
  8. Float - the integer number of carrier phase cycles is not resolved, a float estimate is used instead.
  9. Post processed (PP), data used by the platform is contained within files obtained from an archive such as the IGS archive.
  10. Near real-time (NRT) - data is processed in batch but in a near real-time mode (15 minutes to 1 hour latency).
  11. Real-time (RT) - data is processed in real-time, observations are obtained from the NTRIP caster.
